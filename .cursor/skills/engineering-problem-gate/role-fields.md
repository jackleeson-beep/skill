# Role Fields

本文件定义 `engineering-problem-gate` 的角色扩展字段。  
所有角色都必须先填写主 `SKILL.md` 中的通用字段，再补本文件中的角色字段。

## 数字工程师

- `模块/子系统`
- `接口边界`
- `时序约束`
- `面积/功耗约束`
- `验证状态`
- `依赖模块`

## 模拟工程师

- `电路模块`
- `目标指标`
- `PVT 条件`
- `噪声/失真/稳定性关注点`
- `功耗约束`
- `版图敏感项`

## Layout 工程师

- `版图对象/模块`
- `工艺节点`
- `匹配要求`
- `寄生风险`
- `DRC/LVS 关注点`
- `与电路/封装协同约束`

## 软件工程师

- `输入输出`
- `接口/协议`
- `依赖系统`
- `性能约束`
- `异常场景`
- `验证方式`

## 算法工程师

- `算法目标`
- `输入数据条件`
- `评价指标`
- `延迟/算力/存储约束`
- `训练或推理边界`
- `验证数据来源`

## 测试工程师

- `测试对象`
- `覆盖范围`
- `测试条件`
- `通过准则`
- `风险点`
- `缺陷回归要求`

## 市场产品经理

- `目标用户/客户`
- `使用场景`
- `需求边界`
- `优先级`
- `商业或交付约束`
- `验收标准`

## 现场支持工程师

- `客户现场现象`
- `复现条件`
- `影响范围`
- `当前临时绕过方案`
- `环境限制`
- `反馈时效要求`

## 使用规则

1. 角色字段不替代通用字段。
2. 字段缺失时，明确写 `未知` 或 `待确认`。
3. 如果任务涉及多个角色，先选主角色，再补充相关角色字段。
4. 如果角色字段长期稳定，可继续下沉到 harness 做必填校验。
