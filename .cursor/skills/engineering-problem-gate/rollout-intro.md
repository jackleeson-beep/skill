# Engineering Problem Gate 简介

## 它是什么

`engineering-problem-gate` 是一个面向企业工程团队的 Cursor Skill。  
它的作用不是直接替大家写方案或代码，而是先帮助大家把问题讲清楚、把任务拆清楚，再决定是否进入方案、实现，或沉淀为 `skill / harness`。

## 它解决什么问题

很多工程师在使用 Cursor 时，容易一上来就开始问方案、问实现、问代码，但其实：

- 真正的问题还没定义清楚
- 目标、约束、成功标准没有说清
- 项目背景和资源没有先给 AI
- 任务颗粒太大，AI 很难稳定执行

结果就是：

- AI 回答发散
- 方案不贴项目实际
- 输出看起来很完整，但落地性差
- 团队容易重复试错

`engineering-problem-gate` 的目标，就是把这个过程前置收敛。

## 它怎么工作

它要求工程师在进入下一步之前，先完成四件事：

1. 补齐项目背景和可用资源
2. 用统一模板定义当前问题
3. 用第一性原理做结构化分解
4. 判断是否值得沉淀为 `skill / harness`

## 它带来的价值

- 让 AI 输出更贴近真实项目
- 减少“问题没想清楚就开做”的盲目性
- 提高方案质量和执行效率
- 让不同角色都能用同一套框架协作
- 为后续沉淀企业内 `skill / harness` 打基础

## 适用对象

适用于以下角色：

- 数字工程师
- 模拟工程师
- Layout 工程师
- 软件工程师
- 算法工程师
- 测试工程师
- 市场产品经理
- 现场支持工程师

## 一句话介绍

`engineering-problem-gate` 是一个“先收敛问题、再分解任务、最后才进入执行”的企业级 Cursor Skill，用来减少工程师使用 AI 时的盲目推进，让方案更贴项目、任务更可执行。
