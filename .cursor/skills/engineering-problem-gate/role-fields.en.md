# Role Fields

All roles should fill the common fields first, then add role-specific fields.

## Digital Engineer

- module / subsystem
- interface boundary
- timing constraints
- area / power constraints
- verification status
- dependent modules

## Analog Engineer

- circuit block
- target metrics
- PVT conditions
- noise / distortion / stability concerns
- power constraints
- layout-sensitive factors

## Layout Engineer

- layout block
- process node
- matching requirements
- parasitic risks
- DRC / LVS concerns
- package / circuit co-design constraints

## Software Engineer

- inputs and outputs
- interfaces / protocols
- dependent systems
- performance constraints
- exception scenarios
- validation method

## Algorithm Engineer

- algorithm objective
- input data conditions
- evaluation metrics
- latency / compute / memory constraints
- training or inference boundary
- validation data source

## Test Engineer

- test target
- coverage scope
- test conditions
- pass criteria
- risk points
- regression requirements

## Product / Marketing

- target users / customers
- usage scenario
- requirement boundary
- priority
- business or delivery constraints
- acceptance criteria

## Field Support Engineer

- field symptom
- reproduction conditions
- impact scope
- current workaround
- environment limitations
- response time requirement
