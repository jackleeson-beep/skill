# Engineering Problem Gate (English Companion)

This file is the English companion for `SKILL.md`.

The runtime skill definition remains in `SKILL.md`.  
Use this file as a reference when you want to understand the workflow in English.

## Core Purpose

The skill acts as a project-grade entry gate for engineering work.

It is designed to:

- collect enough project context before execution
- frame the current problem clearly
- decompose work into AI-friendly tasks
- decide whether the task should move to design, implementation, distillation, or further clarification

## What It Is Good At

- project intake before action
- problem framing before solutioning
- structured decomposition before implementation
- deciding whether something is stable enough to become a `skill` or `harness`

## What It Is Not

It is not:

- a final architecture review
- a full project management system
- a replacement for implementation planning
- a substitute for human acceptance

## Workflow Summary

1. Check whether project context is missing
2. Restate the task in one sentence
3. Identify the current phase
4. Fill the common fields
5. Add role-specific fields
6. Split the content into:
   - problem definition
   - solution direction
   - implementation details
7. Decide whether the gate is passed
8. If passed, decompose into smaller AI-friendly subproblems
9. If relevant, decide whether this should become a `skill`, `harness`, or both

## Language Support

- Chinese input -> Chinese output
- English input -> English output
- Mixed input -> use the dominant language and keep important technical terms aligned when useful
